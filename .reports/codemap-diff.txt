Key architectural change summary:
  OLD: Container → chunked HTTP POST → /internal/stream → DO (buffered by CF until response ends)
  NEW: Container → outbound WS → /internal/container-ws → Worker JWT check → DO
       DO.handleContainerWsUpgrade() tags socket, DO.webSocketMessage() routes frames immediately.

  Reason: CF Durable Object HTTP response handler buffers the full body before delivering
  to DO fetch(), making chunked streaming impossible. WebSocket messages are dispatched
  per-frame with no buffering.

  Backward compat: VF_CALLBACK_URL / /internal/stream path retained as fallback.
  claude-agent.js checks VF_WS_CALLBACK_URL first; falls back to HTTP if absent.
